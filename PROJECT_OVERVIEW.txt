                            PROJECT DESCRIPTION
================================================================================

A comprehensive analytics platform for sales data mining, forecasting, and 
visualization. This project implements a complete data analytics pipeline 
from raw data processing through advanced analytics to business intelligence 
integration.

WHAT THIS PROJECT DOES:

This platform transforms raw retail transaction data into actionable business 
insights through a multi-layered analytics approach:

1. DATA WAREHOUSING
   - Converts raw transaction data into a star schema data warehouse
   - Organizes data into fact table (transactions) and dimension tables 
     (customers, products, stores, dates, promotions)
   - Enables efficient analytical queries and reporting
   - Handles large-scale datasets (millions of transactions)

2. EXPLORATORY DATA ANALYSIS
   - Performs comprehensive statistical analysis on warehouse data
   - Identifies sales patterns, trends, and anomalies
   - Analyzes performance across multiple dimensions (products, stores, 
     customers, time periods)
   - Generates data quality reports and visualizations
   - Provides insights into business performance metrics

3. KNOWLEDGE DISCOVERY
   - Implements complete KDD (Knowledge Discovery in Databases) process
   - Discovers hidden patterns and relationships in sales data
   - Applies data mining techniques (clustering, correlation analysis)
   - Generates automated insights and recommendations
   - Creates comprehensive mining reports

4. SALES FORECASTING
   - Predicts future sales trends using time series models
   - Applies ARIMA, SARIMA, and ARMA forecasting techniques
   - Provides forecast confidence intervals
   - Helps with inventory planning and business planning
   - Compares multiple forecasting models for accuracy

5. MARKET BASKET ANALYSIS
   - Discovers which products are frequently bought together
   - Generates association rules for product recommendations
   - Calculates support, confidence, and lift metrics
   - Helps optimize product placement and cross-selling strategies
   - Uses Apriori and FP-Growth algorithms

6. DATA VISUALIZATION
   - Creates static and interactive visualizations
   - Generates dashboards for business intelligence
   - Provides multiple chart types for different analysis needs
   - Enables data export for external reporting tools
   - Supports 3D visualizations for complex data relationships

7. BUSINESS INTELLIGENCE INTEGRATION
   - Exports data in formats compatible with Power BI
   - Embeds Power BI reports directly in the application
   - Provides DAX measures and connection guides
   - Enables seamless integration with enterprise BI tools

8. INTERACTIVE WEB APPLICATION
   - Provides user-friendly web interface built with Streamlit
   - Allows non-technical users to run complex analyses
   - Real-time execution of analytics modules
   - Interactive parameter controls for customization
   - Session-based data management

Key Focus Areas:
- Data Warehouse Design (Star Schema)
- Knowledge Discovery in Databases (KDD Process)
- Time Series Forecasting
- Association Rules Mining
- Advanced Data Visualization
- Power BI Report Embedding
- Interactive Web Application

                         PROJECT ARCHITECTURE
================================================================================

1. DATA LAYER
   Location: data/
   
   Raw Data:
   - data/raw/Retail_Transactions_Dataset.csv
   
   Data Warehouse (Star Schema):
   - data/warehouse/fact_sales.csv (Fact Table)
     * Contains: Transaction_ID, date_id, customer_id, product_id, store_id, 
       promotion_id, Total_Items, Total_Cost, Discount_Applied, Payment_Method, sales_id
   - data/warehouse/dim_customer.csv (Dimension Table)
     * Contains: customer_id, Customer_Name, Customer_Category
   - data/warehouse/dim_date.csv (Dimension Table)
     * Contains: date_id, Date, year, month, day, weekday, season
   - data/warehouse/dim_product.csv (Dimension Table)
     * Contains: product_key, Product
     * Note: Uses product_key as primary key (linked to product_id in fact table)
   - data/warehouse/dim_store.csv (Dimension Table)
     * Contains: store_id, City, Store_Type
   - data/warehouse/dim_promotion.csv (Dimension Table)
     * Contains: promotion_id, Promotion

2. CORE MODULES
   
   kdd_process.py
   - Complete 5-step KDD pipeline implementation
   - Step 1: Selection (data filtering, sampling)
   - Step 2: Preprocessing (missing values, outliers, cleaning)
   - Step 3: Transformation (feature engineering, encoding, normalization)
   - Step 4: Data Mining (patterns, clustering, correlation)
   - Step 5: Interpretation (insights generation, reporting)
   - Output: kdd_results/
   
   time_series_forecasting.py
   - ARIMA (AutoRegressive Integrated Moving Average) model
   - SARIMA (Seasonal ARIMA) model
   - ARMA (AutoRegressive Moving Average) model
   - Stationarity checking (ADF test)
   - Automatic parameter optimization
   - Forecast generation with confidence intervals
   - Model comparison and evaluation
   - Output: forecast_output/
   
   association_rules_mining.py
   - Apriori algorithm for frequent itemset mining
   - FP-Growth algorithm alternative
   - Association rules generation
   - Support, confidence, lift metrics calculation
   - Market basket analysis
   - Insights extraction
   - Output: association_rules_output/
   
   visualizations.py
   - Matplotlib: Static charts and trends
   - Seaborn: Heatmaps and statistical plots
   - Plotly: Interactive dashboards, 3D scatter plots
   - Multiple visualization types:
     * Sales trends over time
     * Category/store/city analysis
     * Correlation heatmaps
     * Interactive dashboards
   - Output: visualizations/
   
   eda_warehouse.py
   - Exploratory Data Analysis on warehouse data
   - Loads all warehouse tables (fact + 5 dimensions)
   - Creates merged dataset for comprehensive analysis
   - Statistical summaries and data quality checks
   - Dimension analysis (customer, date, product, store, promotion)
   - Sales analysis by category, store type, city, season, month, year
   - Correlation analysis and heatmaps
   - Generates multiple visualization charts
   - Handles product_key/product_id column name differences automatically
   - Output: eda_output/

3. APPLICATION LAYER
   
   app.py (Streamlit Web Application)
   - Multi-page interactive GUI
   - 7 main pages:
     1. Home - Project overview and navigation
     2. EDA & Analysis - Data exploration and warehouse analysis
     3. KDD Process - Run complete KDD pipeline
     4. Time Series Forecasting - Generate sales forecasts
     5. Association Rules - Discover frequent patterns
     6. Visualizations - Generate charts and dashboards
     7. Power BI Report - View embedded Power BI report
   
   Features:
   - Interactive controls and parameters for all modules
   - Real-time analysis execution with progress indicators
   - Results visualization with multiple chart types
   - Data export options (CSV, Excel)
   - Objective-focused visualizations (3 key business objectives)
   - Memory-efficient data loading and processing
   - Session state management for data persistence
   - Error handling with user-friendly messages
   - Dynamic column name detection and handling
   - Support for large datasets (16+ million rows)

================================================================================
                         OBJECTIVE-FOCUSED ANALYSIS
================================================================================

The application includes three key business objectives implemented in the Visualizations
page (under KDD Results visualizations section). These objectives use warehouse data 
directly for memory efficiency and accuracy.

OBJECTIVE 1: Product Category Sales Distribution
- Analyze sales by product categories
- Identify high-performing categories
- Compare total cost and item volume
- Top 15 categories visualization
- Performance matrix (scatter plot)
- Uses: fact_sales + dim_product (memory-efficient aggregation)

OBJECTIVE 2: Month-wise Sales Trends and Seasonality
- Monthly sales trend analysis
- Transaction volume tracking
- Seasonal pattern identification
- Combined dual-axis visualizations
- Seasonality distribution charts
- Uses: fact_sales + dim_date (memory-efficient aggregation)

OBJECTIVE 3: Store-Level Sales Performance
- Store performance comparison
- Top 20 stores analysis
- Sales distribution histograms
- Top 10 vs Bottom 10 comparison
- Performance metrics summary
- Uses: fact_sales + dim_store (memory-efficient aggregation)

MEMORY OPTIMIZATION STRATEGY:
All objectives use an optimized approach to handle large datasets:
1. Load fact table and dimension tables separately
2. Aggregate fact table by ID first (reduces memory usage significantly)
3. Merge aggregated results with dimension tables (smaller datasets)
4. This prevents memory allocation errors with large datasets (millions of rows)

DATA SOURCE: Warehouse folder (data/warehouse/) - NOT KDD transformed data
This ensures consistency and accuracy using the source data warehouse.

================================================================================
                         TROUBLESHOOTING GUIDE
================================================================================

COMMON ISSUES & SOLUTIONS:

1. Import Errors
   Solution: pip install -r requirements.txt

2. Memory Errors
   Solution: Use sampling parameters or process in chunks

3. Missing Data Files
   Solution: Ensure data/warehouse/ contains all required CSV files

4. Large Dataset Performance
   Solution: Use sample_size parameter in KDD Process

5. Column Name Errors
   Solution: Code automatically handles column name variations
   - Supports product_key/product_id differences
   - Handles Product_, City_, month column variations
   - Automatic detection after DataFrame operations
   - Robust merge operations with flexible column matching
